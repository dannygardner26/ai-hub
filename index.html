Quick Tips

Claude Code cheat sheet

Practical tips for getting the most out of Claude Code. Copy any command and try it.

Essential
Run shell commands inline
Type ! command in the prompt to run any shell command without leaving the conversation. Output lands right in context.
Use /fast for speed
Toggle /fast for faster output on the same model. Great for quick edits and simple tasks where you don't need deep reasoning.
Escape to cancel, double-Escape to undo
Press Esc to stop generation. Press Esc Esc to undo the last action. Press Ctrl+C to cancel the current input.
Resume with /resume
If Claude gets cut off or you restart, /resume picks up from where you left off with full conversation context.
CLAUDE.md is your instruction file
Drop a CLAUDE.md in your repo root for project-specific rules. Claude reads it at the start of every conversation. Use ~/.claude/CLAUDE.md for global rules.
Prompting
Be specific about what, not how
Say "Add a signup form with email validation and rate limiting" not "Write a function that takes an email string and validates it using regex." Let Claude decide the implementation.
Stack context with @files
Reference files directly: @src/auth.ts look at how auth works here and add OAuth support. Claude reads the file and understands the existing patterns.
Paste screenshots and images
Claude Code is multimodal. Paste a screenshot of a bug, a design mockup, or an error message directly into the prompt. Way faster than describing it.
Start big, then narrow down
"Explain the auth system in this repo" → "Now add OAuth to it" → "Add tests for the OAuth flow." Build Claude's understanding incrementally rather than dumping everything at once.
Tell Claude what NOT to do
"Don't add comments. Don't refactor surrounding code. Don't install new dependencies." Constraints prevent Claude from going overboard with "improvements."
Workflow
Use git worktrees for parallel work
Run multiple Claude instances in separate worktrees for true parallelism: git worktree add ../feature-auth feat/auth. Each gets its own working directory.
Plan before building
Use /plan to enter plan mode. Claude researches, designs, and proposes an approach before writing any code. Prevents wasted effort on wrong directions.
Use subagents for parallel tasks
Claude can spawn background agents for independent tasks. Ask: "Research X while you implement Y." The agents work in parallel and report back.
Review before commit
After making changes, say "review your diff before committing." Claude will catch its own mistakes, remove debug code, and verify correctness.
Let Claude run tests
"Run the tests and fix whatever fails." Claude will execute, read errors, and iterate until tests pass. Don't hand-hold the debug cycle.
Advanced
Hooks for automation
Set up hooks in settings.json to run commands automatically on events. Example: auto-lint after every file edit, auto-test after every commit.
MCP servers extend capabilities
Connect MCP servers for Figma, Playwright, databases, and more. Claude gets direct tool access instead of working through CLI workarounds.
Custom skills are reusable workflows
Create ~/.claude/skills/my-skill/SKILL.md with a name, description, and instructions. Claude auto-triggers the skill when you mention it. Like macros for AI.
Schedule agents with triggers
Use /schedule to create cron-based remote agents that run on a schedule. Great for recurring tasks like daily code reviews or dependency checks.
Memory persists across sessions
"Remember that I prefer Tailwind over styled-components." Claude saves this to its memory system and applies it in future conversations automatically.
Config
Permission modes control safety
Set --allowedTools to control which tools Claude can use without asking. Modes: ask (default), auto-edit (file edits only), full-auto (everything).
Customize the status line
Show token count, model name, or git branch in your terminal status line. Configure in ~/.claude/settings.json under statusLine.
Use /login to switch accounts
Multiple Anthropic accounts? /login lets you switch between them. Use /status to check which account and model you're currently using.
Project-level settings override global
Put a .claude/settings.json in any project for repo-specific tool permissions, MCP servers, and hooks. Overrides your global ~/.claude/settings.json.
